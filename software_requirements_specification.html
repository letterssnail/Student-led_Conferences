Introduction

Background

Student-led conferences present the opportunity for students to lead conferences informing their parents and guardians about their academic progress. Through the program students are able to take control of their learning experience which is a key component to creating an environment in which students can thrive to achieve their best at Surf Coast Secondary College.

Parents and guardians have expressed interest in a redesign of the student-led conference forms and have outlined various design choices that, while meeting the requirements of a booking form, have created an unintuitive and complicated experience.

This specification defines the student-led conferences form that will become a remodel of the student-led conferences form currently in use at Surf Coast Secondary College. Student's at the school are encouraged to attend the conferences on a biannual basis.

Audience

The Form is intended to provide a platform for Guardians and Students to reserve a conference time at the student-led conferences. The Form will have a system that allows Teachers to view the times that their Student's Families have reserved.

To provide a system for the reservation of student-led conference times, a database with User information will be required with booking times associated to Teachers. The Form will provide a platform in which the System Administrator can view, manage and run the Association Database.

Scope

The primary goal of this project is to remodel the user experience of the Original Form, providing a more intuitive and straight-forward experience to Users.

Additional goals of the project are to increase the depth of the Form by providing additional features. The System will incorporate a database that includes additional information on the assocations of Guardians, Teachers and Students which allows Users to reserve a time without the requiring the knownledge of the subjects the Student is undertaking.

Operation of the System

Perspective of the product

To achieve the goals of the project, the program pulls necessary information from the Association Database which is hosted within the School. This product was created after various complaints about the ease and usability of reserving a time for the student-led conferences using the Original Form, the System documented in this specification is intended to replace the existing system.

A context diagram has been developed to highlight the operation of the System:

Features of the product

The Form will provide a simple and functional platform for Guardians and Students to reserve a time for the student-led conferences. Guardians and Students have the ability to reserve times for all Students associated with their Family.

Using their Teacher Login, Teachers will be able to access a list of times reserved under their name. This extends the functionality of the Original Form, this resolution provides clarity for Users.

To organise and oversee the operation of the System, the System Administrator will be able to view and manipulate the Association Database. This specification does not cover the organisation of the System by the System Administrator.

Characteristics and classes of the Users

There are three classes of users for the System: a Family who wishes to reserve a time at the student-led conferences; a Teacher who wishes to view times that are reserved under their name and; the System Administrator who oversees the administration of the Association Database.

This diagram represents the three users operating the System by accessing the Form:

The requirements of technical knowledge that the Users must possess include:

  • All Users must be able to log in to the Form.
  • The Families and Teachers need to possess the technical know-how to operate a computer and browser and navigate to the site that the Form is hosted on.
  • The System Administrator must have sufficient knowledge in operating and the manipulating the Association Database.

Evironment of the System's operation

The System will run via a webserver hosted within the grounds of the School. This ensures that the Association Database will be able to provide sufficient information. This also ensures that the program will rely solely on systems provided at the School, the System can be changed to suit the needs of the School and it will not require external sources that may create an unreliable operating environment.

Design and implementation constraints

The current contraints of the project simply include: the technical limitations of implimenting the System; the ability to implement the System in a school environment; and the technical expertise of the developer.

Documentation for the User

This System is designed to provide an intuitive experience for the User, hence tooltips and manuals will not be included in the user interface. Users are not required to possess former knowledge of the Systems operation or interface. However the Form will include an about page that documents the student-led conferences and their usefulness to the operation of the environment of the School and a sitemap or portal section that links the User to other School projects.

Assumptions and dependencies

Since the System relies upon a webserver that is hosted on the grounds of the School grounds, it relies on the maintenance of the server and the Internet connection that the School is provided with. The Association Database will also be hosted on the server and will be used to reliably send and receive data within the server.

Functional requirements of the System

To represent how processes in new system will handle User information, a diagram showing a graphical representation of the "flow" of data through the new system has been included:

Student-led conferences time reservation

The Form will enable a Family to log in to the System using their Family Login, they can view a list of Students associated with that Family and reserve times associated with those Student's associated Teachers.

Teacher reservation time viewer

Teachers can log in to the System using their Teacher Login, this will enable them to view times that have been reserved under their name and submit break times that will be unreservable by a Family.

System adminisitration and database management

The System Administrator will be able to log in to the System to view and manipulate the Association Database, it is possible for the System Administrator to add and remove Students, Teachers, Guardians and Families.

The user interface of the Form has been designed to create an intuitive visual experience for Users. The design incorporates the Surf Coast Secondary College logo to familiarise the user with the system in use. Experience of the User

The Form intends to direct the User to the required area of retrieval. The user interface does not need to be littered with an array of non-essential options and ideas about the operation and intention of the Form.

Interfaces of the hardware

Since the web form has no designated hardware, it does not have any direct hardware interfaces. The Association Database is managed by the System Administrator via an external computer file and the hardware connection to the database server is managed by the underlying operating system on the web server.

Interfaces of the software

The Form will directly interface with the Association Database, both of which will in turn be interfacing with the operating system and any other software components that is required to run the System.

Interfaces for communication

This System will not utilise any unique communications interfaces.

Inoperable requirements

Requirements of System performance

Although the System does not warrant the need for concern over performance issues, various constraints have been set to reduce the likelihood of system failure or breach of capacity. The maximum response time for the submission of a time reservation request via the Form is 1 minute. The maximum number of time reservation requests is limited only by the amount of Families that are present in the Association Database.

Security concerns of the System

The System will ensure the privacy of information in the Association Database. Viewership of the Association Database or actual alterations, such as removing, changing or adding Users is not possible without authority of the System Administrator since all information in the Association Database is already available to the administration at the School, the database will only be viewable and manipulatable by those with authority.
